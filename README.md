# Secure Network Architecture & Infrastructure Hardening Lab

## Overview
This project documents the deployment and security hardening of a segmented enterprise network environment using CIS Critical Security Controls. The infrastructure was designed to protect administrative systems and server resources while maintaining secure external access.

## Technologies Used
- pfSense Firewall
- Linux Servers
- Network Segmentation
- CIS Security Controls
- System Logging

## Skills Demonstrated
- Network security architecture design
- Firewall rule configuration
- Secure system configuration
- Access control management
- Security monitoring and logging

## Key Activities
- Implemented CIS Critical Security Controls including Secure Configuration, Access Control Management, and Audit Log Management.
- Hardened Linux servers by disabling insecure default configurations and enabling automated security updates.
- Configured pfSense firewall rules to enforce segmentation between AdminNet and ServerNet.
- Implemented least-privilege administrative access across Linux and Windows systems.
- Enabled persistent system logging to support security monitoring and incident investigation.
